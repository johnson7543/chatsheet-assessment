║           Why Do We Have Multiple .env.example Files?           ║
╚══════════════════════════════════════════════════════════════════╝

TLDR: We have 3 .env.example files for 3 different ways to run the app!

┌──────────────────────────────────────────────────────────────────┐
│ 1. ROOT .env.example                                             │
│    Purpose: Docker Compose (containerized full stack)           │
│    Location: ./env.example                                      │
│    Use when: Running "docker-compose up"                        │
│    Contains: ALL variables for both frontend & backend          │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 2. FRONTEND .env.example                                         │
│    Purpose: Local React development                             │
│    Location: frontend/.env.example                              │
│    Use when: Running "npm run dev"                              │
│    Contains: VITE_API_URL=http://localhost:8080                 │
└──────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────┐
│ 3. BACKEND .env.example                                          │
│    Purpose: Local Go development                                │
│    Location: backend/configs/.env.example                       │
│    Use when: Running "go run cmd/api/main.go"                   │
│    Contains: PORT, JWT_SECRET, UNIPILE_API_KEY, etc.           │
└──────────────────────────────────────────────────────────────────┘

═══════════════════════════════════════════════════════════════════

SCENARIO 1: Quick Demo with Docker
───────────────────────────────────
  cp .env.example .env
  docker-compose up -d
  → Visit http://localhost:3000 ✓

SCENARIO 2: Development (Hot Reload)
────────────────────────────────────
  Terminal 1 (Backend):
    cd backend
    cp configs/.env.example configs/.env
    go run cmd/api/main.go
    → Backend on http://localhost:8080 ✓
  
  Terminal 2 (Frontend):
    cd frontend  
    cp .env.example .env
    npm run dev
    → Frontend on http://localhost:5173 ✓

═══════════════════════════════════════════════════════════════════

WHY NOT JUST ONE .env FILE?
────────────────────────────

Different environments need different values:

Local Dev vs Docker:
  • Frontend Port: 5173 (Vite) vs 3000 (nginx)
  • FRONTEND_URL: http://localhost:5173 vs http://localhost:3000
  • Database Path: ./linkedin_connector.db vs /app/data/...

This is a STANDARD practice in monorepos and full-stack apps!

═══════════════════════════════════════════════════════════════════

RECOMMENDATION:
  For Development: Use separate .env files (Scenario 2)
  For Demo/Testing: Use Docker (Scenario 1)
  For Production: Use platform environment variables (not .env files)

See ENV_SETUP_GUIDE.md for complete documentation!
